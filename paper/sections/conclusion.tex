\section{Conclusion}

Sequential falsification is motivated by a simple but important observation: when verifier errors limit the value of resampling, a reasonable next use of inference-time compute is to challenge candidate solutions instead of merely generating more of them. The corrected implementation in this project now supports that idea end to end across function-completion code tasks and stdin/stdout competitive-programming tasks. It also supports answer-only math benchmarks as evaluation targets, but not yet as fully adversarial falsification targets.

The present paper makes three concrete contributions. First, it delivers a working and benchmark-aware falsification pipeline with corrected execution semantics for the major code evaluation suites used here. Second, it replaces several earlier overclaims with a cleaner scientific framing: the current confidence mechanism is a useful empirical ranking signal, but not yet a formally anytime-valid guarantee, and the novelty claim is code-focused rather than a blanket claim over all calibration-aware test-time scaling. Third, it now provides both larger completed runs and monitored partial runs showing that sequential falsification can strongly outperform majority-vote selection on code tasks, with especially large gains on HumanEval+ and meaningful gains on MBPP+ and LiveCodeBench.

The next stage is straightforward but still substantial: finish the resumed HumanEval paper seeds, scale the corrected protocol to the full benchmark counts, integrate stronger external baselines, close the remaining LiveCodeBench and math coverage gaps, and calibrate benchmark-specific false-alarm rates so that the statistical story matches the ambition of the original project. The main scientific hurdle is now clear rather than hidden: falsification already beats agreement-based selection convincingly, but it still needs to separate itself more cleanly from self-debugging and from the strongest local proxy baselines on the hardest HumanEval rows. If those pieces hold up, sequential falsification could become a practical way to push inference-time scaling beyond naive resampling while also producing more trustworthy confidence estimates.
