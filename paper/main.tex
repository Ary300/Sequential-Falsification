\begin{abstract}
Repeated sampling improves code generation only when the downstream verifier is strong enough to suppress false positives. In practice, weak public tests and partial unit tests allow incorrect candidates to survive, which limits the benefit of naive best-of-$N$ scaling. We study \emph{sequential falsification}: after generating a candidate pool, we spend additional test-time compute on targeted probes that try to invalidate candidates before final selection. The resulting system supports HumanEval+, MBPP+, LiveCodeBench, and MATH-style tasks in a single execution framework, but the present falsification claim is deliberately code-focused: math selection is now an oracle-free consensus fallback until a real math verifier is integrated. Our current implementation uses a testing-inspired multiplicative confidence proxy and evaluates calibration empirically, rather than claiming a fully anytime-valid e-process without externally calibrated error rates. Across the strongest DeltaAI code runs currently available, sequential falsification consistently beats majority vote and often approaches the oracle: on monitored HumanEval+ runs it improves from $34.5 \pm 0.3$ to $79.6 \pm 1.5$ on DeepSeek-R1-Distill-Qwen-32B versus an oracle of $83.5 \pm 0.6$, from $32.3$ to $65.2$ on DeepSeek-R1-Distill-Qwen-7B versus an oracle of $72.6$, and from $23.8 \pm 1.2$ to $42.4 \pm 1.5$ on Qwen2.5-Coder-14B versus an oracle of $43.6 \pm 1.5$. On completed MBPP+ runs, it reaches $33.8 \pm 1.0$, $75.1 \pm 0.3$, and $64.4 \pm 1.6$ for the 7B, 14B, and 32B models respectively, while a corrected 100-problem LiveCodeBench v6 run improves from 3.0 to 8.0. The baseline story is intentionally harder than in earlier drafts: the current bundle also includes a non-adaptive generated-test filter and executable local CodeT/S* proxy baselines, which expose mixed but informative comparisons rather than strawman wins. Self-debug remains a strong baseline and is still competitive or slightly better on several current rows, so the present empirical contribution is strongest as a diagnosis of agreement-based failure and a calibrated elimination framework rather than a universal accuracy win over every sequential baseline.
\end{abstract}
